## ACM

### [D - Minimize Range](https://atcoder.jp/contests/abc450/tasks/abc450_d)

- 给定长度为 `N` 的数组 `A` 和正整数 `K`。
  你可以**任意次**给数组中某个数加上 `K`。
  求操作后，数组最大值减最小值的**最小可能值**。
- 结合 AtCoder D 题的风格以及复杂度，这题看起来就是一个贪心。
- 首先不难想到的是：我们可以把所有比最大值小的数往上加，直到逼近某个上界。
- 然后排序后，我们会得到 `val1-valn`，并且它们的差值不会超过 `K`。
- 其实我们的第一个可能答案是 `valn-val1`。
- 通过观察或者说手玩，我们可以选择给最小的 `val1 += k`，于是答案就会转化成 `val1+k-val2`。
- 随后不断操作，就继续给 `val2 += k`，然后答案就会继续轮转。
- 本质上是一个序列的轮转，比如开始是 `12345`，操作后依次变为 `23451 -> 34512 -> 45123 -> 51234 -> 12345`。

### [F - Strongly Connected 2](https://atcoder.jp/contests/abc450/tasks/abc450_f)

#### 题意转化

首先需要注意一点，这题本质上不是图论题。因为题目只给出了 `n-1` 的线路，所以强联通的要求可以转化成：从 `1` 开始最终走到 `n`。然后进一步看，对于所有的边来说，都等效于连接了一段区间。参考样例 `1`：

#### DP 定义

考虑 DP，定义 \(dp_i\) 代表从 `1` 这个点最终走到 `i` 这个位置的方案数量，初始值是 \(dp_1 = 1\)。转移是依赖于边的，所以维护 \(dp_i\) 的时候，只需要使用终点是 `i` 的边。

#### 转移理解

枚举终点 `j`，这个时候考虑所有 `u -> j` 的边对 DP 的影响。首先我们可以统计 \(dp_1 \sim dp_{j-1}\) 的和，这个和表示前面所有终点小于 `j` 的边所形成的方案数量，记为 `sum`。那么显然，假设终点为 `j` 的边有 `cnt` 个，那么枚举后，所有的情况应该是 \(sum \cdot 2^{cnt}\)。

那么两个问题，如何求 \(dp_j\) 呢？这里使用容斥：只要求出所有更新后 \(dp_1 \sim dp_{j-1}\) 的方案数就可以了。那么考虑一条边对 \(dp_1 \sim dp_{j-1}\) 的影响。假设这条边是 `[k, j]`，那么显然对于所有的 \(dp_1 \sim dp_{k-1}\)，这条边是可选可不选的，所以这个区间需要乘 `2`；而剩下的区间一旦选了就会直接到 `j`，因此 \(dp_j\) 我们单独用容斥更新，这里不需要重复考虑。

#### 做法总结

总结下来就是，我们需要支持前缀 DP 求和，以及维护一个区间 `*2` 的操作。考虑使用线段树，然后进行简单的 DP 转移就行了。

### [E - A += v](https://atcoder.jp/contests/abc449/tasks/abc449_e)

- 个人感觉是一道比较难做的题，比 `449D` 更难搞一点。
- 首先观察出本质上整个加数过程分成两个进程，第一个进程是不断把所有小的数同步到次数第 `i` 大的数。
- 参考样例 `1`：`1 1 2 -> 1 1 2 3 -> 1 1 2 3 2 -> 1 1 2 3 2 3`。先是 `3` 加到和 `2` 一样的次数，然后 `2, 3` 都加到和 `1` 一样的次数。
- 我们记录第一个进程为 `pre` 次，第 `i` 大出现的次数是 `cnt[i]`。
- 第二个进程是所有数都和原始的最大次数相同后，就变成一个不断加 \(1, 2, 3, 4, \dots, m\) 的过程。
- 那么显然对于 `x <= n` 以及 `x > pre` 都是比较好做的。
- 问题出现在第一个进程中，首先我们计算出来每一个进程需要加多少个数，然后可以定位到操作是在同步第几大的次数。
- 那么在同步第 `x` 大次数的时候，我们加的轮数应该是 `cnt[x] - cnt[x-1]`，就是样例 `1` 中 `1 1   1  2 3` 这种情况，也就是 `2, 3` 需要加两轮。
- 所以我们计算出是那一轮的第几个数，记为 `rk`。
- 这个时候问题就变成了：我们需要找到所有 `cnt[x] < cnt[id]` 的数中，第 `rk` 小的那个数字（假设目前同步的是 `id` 大的数）。
- 这本质上是一个第 `k` 小问题，考虑线段树维护：对所有的查询进行离散化，然后从小到大开始枚举。每次查询同步 `id` 大的数的时候，把前面的数做单点修改就好。比如我们有 `4 4 4 1 1 2`，这里假设是同步 `cnt[1]`，那么我们就要把 `2, 3` 都单点修一下，这个时候第一小就是 `2`，第二小就是 `3`。然后在同步 `cnt[4]` 的时候，`1` 也加进来，那么这个时候对应的是 `1, 2, 3`；如果从线段树的视角看，就是 `0 1 1 0 -> 1 1 1 0`。
