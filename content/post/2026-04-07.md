## [E - Laser Takahashi](https://atcoder.jp/contests/abc442/tasks/abc442_e)

- 感觉 key-code 就是一个 polar angle sort，排序后变成一个环状区间和问题。
- 先特判 \(l\) 和 \(r\) 是不是一条直线上，然后分讨：\(l < r\) 直接前缀和，\(l > r\) 说明扫过了端点，交换 \(l, r\)，改求没有被扫到的地方。
- 几个小细节：
  - 点记得除 \(\gcd(x, y)\)，并且取绝对值。
  - 本题是顺时针转动。

```cpp
inline bool up (point p) {
  return p.y > 0 or (p.y == 0 and p.x >= 0);
}

sort(v.begin(), v.end(), [] (point a, point b) {
  return up(a) == up(b) ? a.x * b.y > a.y * b.x : up(a) < up(b);
});
```

## [F - Diagonal Separation 2](https://atcoder.jp/contests/abc442/tasks/abc442_f)

- ABC 经常喜欢靠这种神秘黑白 + 二维格子的题。
- 首先这种题都是优先考虑 \(dp\)，如果一眼看不出和什么最短路相关的话，那么常见的肯定是 \(dp_{i,j}\)。
- 这里注意到 \(N = 5000\)，那么显然 \(N^3\) 是不够的。但其实 \(N^3\) 的状态设计个人觉得是比较好想的。
- 定义 \(dp_{i,j}\) 为到第 \(i\) 行并且这一行第 \(j\) 列是白色时，合法情况的最小操作数量。（题目的两个约束经过分析就是白色每行每列的数量不增加。）
- 那么显然在更新 \(dp_{i,j}\) 的时候，可以枚举上一行的 \(dp_{i-1,k}\)，其中 \(k \ge j\)。因为上一行是没有影响的。
- 然后这样是 \(N^3\) 的，发现枚举的 \(k\) 可以使用一个后缀数组优化，\(N^2\) 跑过去。
