## ACM

### [P4513 小白逛公园 - 洛谷](https://www.luogu.com.cn/problem/P4513)

- 这题就是一个很经典的区间最大子段和线段树。
- 相比普通的区间和模板，这题的关键不在线段树本身，而在于一个节点到底要维护什么信息。
- 这里其实也很符合我前两天在想的东西：线段树的本质，还是看儿子怎么正确合并到父亲。

#### 维护的信息

- 对于一个区间，我们维护四个量：
  - `sum`：整段区间和
  - `pre`：最大前缀和
  - `suf`：最大后缀和
  - `ans`：最大子段和
- 其中 `sum` 是最基础的拼接信息，`pre` 和 `suf` 是为了处理中点两边拼起来的情况，`ans` 则是这个区间真正的答案。

#### 为什么四个量就够了

- 因为一个区间的最大子段和只可能来自三种情况：
  - 完全在左儿子内部
  - 完全在右儿子内部
  - 横跨中点，也就是左区间的最大后缀和加上右区间的最大前缀和
- 所以父亲区间的答案就是：

\[
ans = \max\bigl(l.ans,\ r.ans,\ l.suf + r.pre\bigr)
\]

- 然后整段和显然是：

\[
sum = l.sum + r.sum
\]

- 最大前缀和要么完全在左边，要么是整个左区间再接上右区间的一段前缀：

\[
pre = \max\bigl(l.pre,\ l.sum + r.pre\bigr)
\]

- 最大后缀和同理：

\[
suf = \max\bigl(r.suf,\ r.sum + l.suf\bigr)
\]

- 所以只要把这四个量维护好，父亲节点就可以完全由两个儿子推出。

#### 和代码的对应

- 我代码里写了一个 `info` 结构体，里面正好就是这四个量。
- 然后直接重载 `operator +`，把两个儿子的 `info` 合成父亲的 `info`。这样 `build`、`change`、`query` 都会比较自然。
- 这里还有一个很关键的细节：叶子节点初始化的时候，四个值都必须直接等于这个点本身的权值。

\[
info(x) = \{sum = x,\ pre = x,\ suf = x,\ ans = x\}
\]

- 不能写成 `max(x,0)`，因为这题不允许你选空区间。也就是说如果整个区间全是负数，那么答案就是其中最大的那个负数，而不是 `0`。

#### 查询为什么也要返回结构体

- 这题查询的不是简单一个区间和。
- 但是你如果只在查询时返回 `ans`，那是没法继续往上合并的，因为跨中点的时候你还需要 `pre` 和 `suf`。
- 所以查询函数必须返回整个 `info`，最后再取 `.ans` 才行。

#### 一点实现细节

- 这题修改是单点修改，所以不需要懒标记，正常递归到底修改叶子再 `updata` 回来就行。
- 题目里查询会出现 `a > b`，所以记得先交换。
- 还有一个很容易忽略的点是：这个合并不是交换律的，`left + right` 和 `right + left` 不是一回事，所以查询时左右区间的合并顺序不能写反。
- 代码里 `pre` 和 `suf` 写成了带 `max(0ll, ...)` 的形式，本质上和标准写法是等价的，核心还是那套四元组合并。

#### 做法小结

- 总结下来，这题其实就是线段树维护“可合并区间信息”的一个非常标准的板子题。
- 真正要理解的不是线段树递归本身，而是为什么区间最大子段和可以拆成 `sum / pre / suf / ans` 这四个量。
- 只要把这四个量之间的合并关系想明白，这题就基本结束了。

### [P1471 方差 - 洛谷](https://www.luogu.com.cn/problem/P1471)

- 这题也是一个很典的线段树题，不过相比前面那题，这里更重要的是把数学式子先整理对。
- 题目要求区间加、区间平均数、区间方差，所以很自然地会想到维护区间和；但如果只维护区间和，那么方差是没法直接求的。
- 这里的关键点就在于：方差可以拆成二次项和一次项。

#### 方差公式变形

- 设一段区间长度是 \(len\)，区间和是 \(S = \sum a_i\)，平方和是 \(Q = \sum a_i^2\)。
- 那么平均数就是：

\[
\overline a = \frac{S}{len}
\]

- 方差定义是：

\[
s^2 = \frac{1}{len}\sum (a_i-\overline a)^2
\]

- 把它展开以后，其实就是：

\[
s^2 = \frac{Q}{len} - \left(\frac{S}{len}\right)^2
\]

- 所以这题本质上只要维护两个东西：
  - 区间和 `sum`
  - 区间平方和 `sq`

#### 区间加怎么更新

- 真正难一点的是区间加 `k` 的时候，平方和怎么变。
- 假设原来这一段每个数都加上 `k`，那么：

\[
(a_i + k)^2 = a_i^2 + 2ka_i + k^2
\]

- 对整个区间求和，就得到：

\[
Q' = Q + 2kS + len \cdot k^2
\]

- 同时区间和会变成：

\[
S' = S + len \cdot k
\]

- 所以这题的线段树节点信息其实很简单，就是 `sum`、`sq` 和一个区间加的懒标记 `tag`。
- 下传和区间修改时，核心就是套上面这两个式子。
- 这里还要注意顺序：更新平方和的时候用到的是修改前的 `sum`，所以代码里一般要先存旧值，或者按正确顺序更新。

#### 查询怎么做

- 查询平均数的时候，直接返回区间和 `sum`，再除以区间长度即可。
- 查询方差的时候，返回这一段的 `sum` 和 `sq`，然后套公式：

\[
ans = \frac{sq}{len} - \left(\frac{sum}{len}\right)^2
\]

- 这样就不需要真的去算每个点减去平均数后的平方了。

#### 做法小结

- 这题和前面最大子段和那题的区别在于：前者是维护“可合并结构”，这题则更偏向“维护数学统计量”。
- 但是本质没变，还是线段树每个节点维护一组足够的信息，让父亲能由儿子合并出来。
- 只要先把方差公式整理成 `平方和 - 平均数平方` 这一步，后面基本就是标准区间加线段树了。

### [P1558 色板游戏 - 洛谷](https://www.luogu.com.cn/problem/P1558)

- 这题也是一个很标准的线段树题，不过这次维护的不是最值也不是区间和，而是“这一段出现过哪些颜色”。
- 题目有两个操作：
  - 把一整段全部染成某一种颜色
  - 查询一段区间里出现了多少种颜色
- 看到 `T \le 30` 这个范围，其实就已经很明显了：可以直接用一个二进制状态压颜色集合。

#### 维护的信息

- 对于每个节点，我们维护一个二进制掩码 `mask`。
- 如果这一段里出现了颜色 `c`，那么就让第 `c-1` 位为 `1`。
- 比如只有颜色 `1`，那就是：

\[
mask = 1 \ll 0
\]

- 如果同时出现颜色 `1, 3, 4`，那对应的状态就是把第 `0, 2, 3` 位都设成 `1`。
- 这样一来，一个区间里出现过哪些颜色，就可以直接用一个整数表示出来。

#### 父亲怎么由儿子合并

- 这个合并就比最大子段和那题简单很多了。
- 因为左区间出现过的颜色，和右区间出现过的颜色，合起来就是整段区间出现过的颜色。
- 所以父亲节点的信息就是：

\[
mask = l.mask \ |\  r.mask
\]

- 本质上就是一个按位或。

#### 区间染色怎么处理

- 这题修改不是区间加，而是区间覆盖。
- 假设把一整段都染成颜色 `c`，那么这一整段的颜色集合就会被直接重置成只有这一种颜色：

\[
mask = 1 \ll (c-1)
\]

- 所以这题的懒标记也很好设计：直接记录“这一段被整体改成了哪一种颜色”即可。
- 下传的时候，不是叠加，而是覆盖，因为后来的染色会把前面的颜色全部冲掉。
- 这点和区间加法那类题不一样，这里标记之间的关系不是累加，而是直接替换。

#### 查询怎么做

- 查询一段区间时，返回这一段的 `mask`。
- 如果查询跨左右儿子，就把两个结果按位或起来。
- 最后答案就是这个二进制状态里有多少个 `1`，也就是 `popcount(mask)`。

#### 一点实现细节

- 初始整块色板颜色都是 `1`，所以建树时所有叶子节点初始值都是 `1 << 0`。
- 题目里同样会出现 `A > B`，记得先交换。
- 因为 `T \le 30`，所以用一个 `int` 就够了；如果写得保守一点，用 `long long` 也完全没问题。

#### 做法小结

- 这题本质上就是“区间覆盖 + 区间并集查询”的线段树。
- 真正的关键只有两个：
  - 用二进制位表示颜色集合
  - 用覆盖型懒标记处理区间染色
- 这样修改是区间覆盖，查询是区间按位或，最后统计二进制里 `1` 的个数就结束了。
