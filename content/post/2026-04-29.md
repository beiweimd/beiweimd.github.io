## ABC413 D / E

### [D - Make Geometric Sequence](https://atcoder.jp/contests/abc413/tasks/abc413_d)

- 题意就是问能不能把数组重排成一个等比数列。
- 这里最关键的一点是：不要真的去算公比，直接用交叉相乘判断三项是否满足等比关系：

$$
a_i \cdot a_{i+2} = a_{i+1}^2
$$

- 因为题目保证 $A_i \ne 0$，所以普通情况下可以先按绝对值从大到小排序。这样如果存在某种等比顺序，绝对值这一层基本会被固定下来，然后只需要顺着检查上面的式子即可。
- 代码里用的是：

```cpp
sort(A.begin(), A.end(), [](const ll &a, const ll &b) {
    return abs(a) > abs(b);
});

for (int l = 0; l < n - 2; l++) {
    if (A[l] * A[l + 2] != A[l + 1] * A[l + 1]) {
        flag = 0;
        break;
    }
}
```

- 另外要单独处理 $|r|=1$ 的情况，也就是所有数的绝对值都相等：
  - 如果全是同一个数，那么公比是 $1$，一定可以。
  - 如果只有 $x$ 和 $-x$，那么想用公比 $-1$ 交替排列，两个符号的数量差不能超过 $1$。
- 所以代码里先用 `map` 统计正负数量，遇到同绝对值的情况就提前判掉。这个特判其实挺重要，不然单纯按绝对值排序时，符号排列不一定自然满足交替。

### [E - Reverse 2^i](https://atcoder.jp/contests/abc413/tasks/abc413_e)

- 操作是翻转长度为 $2^i$ 的对齐区间，要求在所有可达排列里求字典序最小的那个。
- 这个题的核心直觉是递归地看区间：一个长度为 $2^k$ 的区间，可以拆成左右两个长度为 $2^{k-1}$ 的子区间。子区间内部先各自变成能达到的最小形态，然后当前层只需要决定：

```text
左半放前面，还是右半放前面
```

- 所以你的 `build(l, r)` 做的事情就是：
  1. 先递归处理左半。
  2. 再递归处理右半。
  3. 比较左右半的字典序。
  4. 如果左半更大，就把两个半区交换，让更小的半区排在前面。

关键部分是这段：

```cpp
int lpos = l, rpos = mid + 1;
while (lpos <= mid && a[lpos] == a[rpos]) {
    lpos++;
    rpos++;
}
if (lpos == mid + 1) return;

if (a[lpos] > a[rpos]) {
    lpos = mid;
    rpos = r;
    while (lpos >= l) {
        swap(a[lpos], a[rpos]);
        lpos--;
        rpos--;
    }
}
```

- 前半段是在比较左右半区的字典序。
- 后半段看起来像是反着 `swap`，但实际效果是把左右两个半区整体交换，并保持各自内部顺序不变。

比如：

```text
[a b | c d] -> [c d | a b]
```

- 这样从小区间一路合并到大区间，每一层都把字典序更小的半区放到前面，最后整段就是可达状态里的最小字典序。
- 复杂度上，每一层所有区间加起来处理 $O(2^n)$ 个元素，一共有 $n$ 层，所以总复杂度是：

$$
O(n \cdot 2^n)
$$

## 5.1

- 扩欧-同余方程。
- 密码学的课上听了下，回去又看了下。也算是厚积薄发？之前阶段看的觉得搞不懂或者是比较hard的东西现在看起来还是十分简单直观的。

```cpp
void exgcd(ll a, ll b, ll &x, ll &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return;
    } else {
        exgcd(b, a % b, x, y);
        ll t = x;
        x = y;
        y = t - (a / b) * y;
    }
}
```

- 给出扩欧模板，本质上就是在求解 $\gcd(a,b)=\gcd(b,a\bmod b)=c$ 的同时，维护了 $ax+by=c$ 这个事情。
- 在 gcd 递归到 $(a,0)$ 的时候存在 $a \cdot 1 + b \cdot 0 = c$。
- 然后反向递归计算新的 $x,y$ 就好了。

![1777635389672](/post/image/2026-04-29/1777635389672.png)

- 基于此，我们可以求解 $a$ 的逆元，前提是 $\gcd(a,\mathrm{mod})=1$，不然不定方程无解：

$$
ax=\mathrm{mod}\cdot k+1 \Rightarrow ax-\mathrm{mod}\cdot k=1
$$

## 解不定方程：

- 首先通过上面我们知道求解 $ax+by=d$ 是有解得，当且仅当 $d$ 是 $\gcd(a,b)$，后文记为 $c$ 的倍数，即 $c\mid d$。
- 我们通过求解得结果满足 $ax+by=c$，要变化一下，然后求出得仅仅是一组特解，实际上这种东西本质上是在 $\bmod d$ 上变换的，所以有很多解，我们可以对 $x$ 进行加，$y$ 进行减然后变换。
